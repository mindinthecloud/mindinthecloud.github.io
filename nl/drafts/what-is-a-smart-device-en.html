What is a ‘smart’ device?

2014/09/09

“This hospital bed is equipped with a smart device that stores all the patient’s data.” Just a phrase in a discussion about an imaginary case of interconnected devices, also know as ‘The Internet of Things’. This made me wonder: what is a ‘Smart Device’?

In this article I’ll try to create some clarity around this concept.

From the context one thing was clear: a smart device is just a ‘thing’ in the internet of things. A just slightly funny game of words that adds little to a better understanding of the concept. But the context of ‘internet’ provides us at least with a first question to answer.

Is a ‘smart device’ internetworked by itself?
Should it have it’s own IP network stack?

I tend to answer this by defining a smart device as accessible from the internet, be it by it’s own IP stack or by means of some kind of gateway or bridge. This implies that one can transform a ‘dumb device’ into a smart one by simply adding some kind of electronic circuitry that provides it with an IP network stack. This has been done for far over a decade by equipment called terminal servers (the traditional type, not the kind that made Citrix a big player in the corporate world). These devices provided access over IP to equipment that only had serial interfaces conforming to standards like RS-232 or RS-422.

With only this definition a simple sensor —like a thermometer— equipped with a serial interface and connected to a terminal server would qualify as a ‘smart device’. We need to add another aspect.

What makes a device ‘smart’?
…besides being internetworked?

For this aspect I would like to add two characteristics for a device to qualify as being ‘smart’:

  1. its functionality can be changed without any physical contact
  2. its functionality can be extended or modified beyond what was previously available

Take for instance an IP-connected videocamera. The functionality of such a device can be modified, by nature the IP network stack at least. Maybe the focus can be adjusted as well, or it can tilt and pan on command. But I would not qualify is as smart, because its functionality is fixed. At least as long as there is no means of remotely loading it with new software. If the software code is ‘frozen’ in read-only memory and the device itself provides no means of altering the code without a physical connection I would consider such a device as merely ‘programmable’.

A game console on the other hand can very well be considered ‘smart’. It is provided with writeable memory that can be loaded with functionality (programs) that can be retrieved from a remote location. It gets a little slippery here; ane can imagine edge cases like game consoles that only accept memory cards but have no network stack. But those are not internetworked and are disqualified from the ‘internet of things’.
